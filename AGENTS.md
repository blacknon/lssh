AGENTS.md
===

## 概要

- このリポジトリは `lssh` を中心とした SSH クライアントスイートです。
- CLI エントリポイントは `cmd/*` にあり、実装の中心は `internal/app/*` と関連する `internal/*` にあります。
- 単一コマンドの修正でも、共有処理に触れる場合は他コマンドへの影響を必ず確認してください。

## 作業

- Go のバージョンは `go.mod` に合わせてください。古い Go への互換対応を勝手に追加しないでください。
- 既存のディレクトリ構成とコマンド命名 (`lssh`, `lscp`, `lsftp`, `lssync`, `lsdiff`, `lsshfs`, `lsmon`, `lsshell`, `lsmux`, `lspipe`) を維持してください。
- `github.com/kevinburke/ssh_config` は upstream に戻さず、`./internal/ssh_config` の置き換え前提を保ってください。

## 実装方針

- 変更範囲はできるだけ局所化し、無関係なリファクタは混ぜないでください。
- CLI フラグ、設定ファイル形式、既存のユーザー向け挙動を不必要に変えないでください。
- クロスプラットフォーム性を意識してください。特に Linux / macOS / Windows の差異がある処理では、対象 OS を明示して実装してください。
- `lsshfs` まわりは OS 制約があるため、Linux/macOS 前提の挙動を他コマンドに波及させないでください。
- TUI 操作や複数ホスト処理では、1台向けの変更が並列実行時に破綻しないか確認してください。
- 基本的に、コマンド実行をユーザが指定する機能以外ではOSコマンドを実行しての機能実装はしないでください。もしそれしか実現方法がない場合は都度聞いてください。

## ビルドとテスト

- 変更後は、まず影響範囲に近いテストを優先して実行してください。
- 最終確認として可能なら `make test` を実行してください。
- ビルド確認が必要な変更では `make build` を実行してください。
- 変更対象が限定的な場合は、`mise.toml` にある個別タスクや該当パッケージの `go test` を優先して構いません。
- テストやビルドが未実行なら、実行していないことと理由を最終報告に明記してください。

## ドキュメント更新ルール

- ユーザー向け挙動、CLI オプション、設定方法、インストール手順を変えた場合は、対応する README / docs も更新してください。
- 主に次を確認してください。
  - `README.md`
  - `docs/*.md`
  - 影響を受ける `cmd/*/README.md`

## 変更時の注意

- デフォルト値や設定キー名を変える場合は、既存ユーザーへの影響を最小化してください。
- 破壊的変更を入れる場合は、移行手順または互換レイヤを検討してください。
- 新規依存の追加は、本当に必要な場合だけにしてください。
- セキュリティや接続処理に関わる変更では、proxy / forwarding / auth への副作用を確認してください。

## 最終報告の書き方

- 何を変えたかを簡潔に述べてください。
- 実行したテストやビルドコマンドを列挙してください。
- 未確認事項、既知の制約、OS 依存の注意点があれば明記してください。
