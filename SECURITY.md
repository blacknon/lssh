# Security Policy

## Supported Versions

Security fixes are provided for the latest released version of `lssh` and the current default branch.

Older releases may not receive backported fixes.
If you are not running the latest release, please upgrade first and confirm whether the issue still reproduces.

## Reporting a Vulnerability

Please do not open a public GitHub issue for suspected security vulnerabilities.

If GitHub Private Vulnerability Reporting is enabled for this repository, use it to report the issue privately.

If private reporting is not available, contact the maintainer directly at `blacknon@orebibou.com`.

## What To Include

Please include as much of the following as you can:

- A description of the issue and the expected impact
- Steps to reproduce the problem
- The affected command or feature, such as `lssh`, `lscp`, or `lssync`
- Version information, OS, and any relevant configuration details
- A proof of concept, logs, or screenshots if they help explain the issue
- Any mitigation ideas or suggested fix, if available

## Disclosure Process

We will try to acknowledge the report promptly, investigate it, and coordinate a fix before public disclosure.

After a fix is available, the details may be published through GitHub Security Advisories, release notes, or repository documentation.

## Scope

This policy applies to the code in this repository.
For vulnerabilities in third-party dependencies, please also consider reporting them to the upstream project.
